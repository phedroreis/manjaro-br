Procurei ler os seus log's!

ID do artigo: 93
Adaptado por: lobocode
Adaptado em: 06 Jun 2013, 16:18
Descrição do artigo: O texto abaixo mostra a importância de se ler os log's gerados pelo sistema.
Link externo para este artigo:
kb.html?a=93


É muito comum ver usuário pedindo ajuda para solucionar um problema no sistema ( o que é natural ). Aprender a verificar, ler arquivos de log's é essencial para você que deseja ir direto ao ponto, ao erro sem a necessidade de contar com anos de experiência na área ou ficar deduzindo/especulando qual possa ser o problema. Até porque, é comum achar que o erro é X e descobrir que na verdade é Y e por fim, acaba gerando um novo problema por ter feito tudo errado.

A atividade dos programas são registradas em arquivos localizados em /var/log . Estes arquivos de registros são chamados de logs e contém a data, hora e a mensagem emitida pelo programa (violações do sistema, mensagens de erro, alerta e outros eventos) entre outros campos. Enfim, muitos detalhes úteis ao administrador tanto para acompanhar o funcionamento do seu sistema, comportamento dos programas ou ajudar na solução e prevenção de problemas.

Além disso, muitos outros programas geram logs em subpastas do /var/log/ para não se misturarem com os logs do sistema. Uma dica bacana para você descobrir para onde vai o log da autenticação do sistema, do kernel usado, das mensagens de erro e outros, é verificando o conteúdo do arquivo /etc/syslog-ng/syslog-ng.conf. Exemplo:

Código: Selecionar todos
....
..........
destination d_authlog { file("/var/log/auth.log"); };
destination d_syslog { file("/var/log/syslog.log"); };
destination d_cron { file("/var/log/crond.log"); };
destination d_daemon { file("/var/log/daemon.log"); };
destination d_kernel { file("/var/log/kernel.log"); };
destination d_lpr { file("/var/log/lpr.log"); };
destination d_user { file("/var/log/user.log"); };
destination d_uucp { file("/var/log/uucp.log"); };
destination d_mail { file("/var/log/mail.log"); };
destination d_news { file("/var/log/news.log"); };
destination d_ppp { file("/var/log/ppp.log"); };
destination d_debug { file("/var/log/debug.log"); };
destination d_messages { file("/var/log/messages.log"); };
destination d_errors { file("/var/log/errors.log"); };
destination d_everything { file("/var/log/everything.log"); };
destination d_iptables { file("/var/log/iptables.log"); };
destination d_acpid { file("/var/log/acpid.log"); };
destination d_console { usertty("root"); };
...........


Observe que este arquivo lhe diz onde estão os log's que você necessita para verificar se há falhas no sistema. Essa é uma das principais dicas para quem está começando e principalmente com quem já trabalha com "servidores" como também é claro, com qualquer tipo de sistema "Unix/Linux".

Aprenda a ler log's e nos ajude a te ajudar quando você necessitar de suporte para seu sistema Manjaro.
